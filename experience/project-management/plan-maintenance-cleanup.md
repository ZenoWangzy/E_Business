# 项目计划维护与清理

## ✅ 推荐模式

**定期清理已完成项目** - 保持计划文档简洁，聚焦待办项

```bash
# 文件: plan.md
# ✅ 完成后删除详细实施方案，只保留执行日志记录
- ✅ 问题1：事务间隙 —— 已完成（见执行日志 2026-01-03）

# ✅ 在执行日志中保留完整记录
## 执行日志
### 2026-01-03
- ✅ 问题1-18：全部完成修复
- 📋 待办：5个Low级别优化项
```

```bash
# ✅ 使用head命令精确清理
head -n 358 plan.md > /tmp/plan_cleaned.md && mv /tmp/plan_cleaned.md plan.md
```

## ❌ 反模式

```markdown
# ❌ 保留所有已完成问题的详细代码示例
## 问题1: 事务间隙 (100+行代码)
## 问题2: 孤儿文件 (80+行代码)
# ... 文件持续膨胀到466行
```

```bash
# ❌ 不清理已完成项
# ❌ 手动Edit删除大段内容（容易出错）
# ❌ 不更新时间表和置信度
```

## 💡 核心原则

- **只保留待办项**: 已完成项移到执行日志，删除详细实施方案
- **精确行数清理**: 使用 `head -n N` 而非手动Edit
- **动态更新**: 清理后更新时间表（8周→2周）和置信度
- **极简主义**: 文件从466行精简到358行，提高可读性
